## After each profile goes live

Append a row to `data/backlinks/inventory.json`:

```json
{
  "id": "directory-<slug>",
  "bucket": "directory",
  "target_url": "<the exact URL of the new profile>",
  "status": "live",
  "live_url": "<same URL>",
  "first_seen": "<YYYY-MM-DD>"
}
```

Then re-export Bing referring-domains → `data/backlinks/bing-export-YYYY-MM-DD.csv` →
`npm run backlinks:snapshot <path>` → `npm run backlinks:verify` to see the
gap close.

## Note on rate of return

A typical pattern for these 14:

- 10–14 will accept the listing
- 6–10 will be live within a week of submission (immediate publish or quick review)
- 4–6 may take 1–4 weeks to go through editorial review
- 1–2 will be rejected or stalled

That means realistically: **8–12 directory backlinks added to the snapshot
within 30 days** for ~2 hours of your time.

Combined with the awesome-list PRs (5 already open, ~5 more queued),
redistribution bundles (7 destinations ready), and the embed/oEmbed
machinery, we're plausibly at **30–60 new referring domains in the first
quarter** — a meaningful step toward the 1000-domain Goal.
